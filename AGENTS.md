# AGENTS.md

## Project rules

- Build a local-first MVP for a Hegemony board-game assistant.
- Backend is the single source of truth for game state and rules.
- Keep domain logic framework-independent and outside Spring-specific layers.
- Do not use JavaFX, Swing, or Vaadin.
- Frontend must be React + TypeScript + Tailwind + shadcn/ui.
- Prefer a working vertical slice over broad incomplete scaffolding.
- Do not implement the full board game yet.
- Add only a minimal demonstrative rules subset for the first version.
- The bot must choose only from legal moves generated by the engine.
- Use JSON save/load only for persistence in the first iteration.
- Write tests for rule validation and state transitions.
- Keep code explicit, readable, and easy to extend with future card JSON logic.

## Definition of done for early iterations

- App runs locally
- User can submit a structured move
- Backend validates and applies it
- Bot can choose and explain a legal response move
- UI clearly shows board state and event log
- Tests pass

## Editing rules
- Prefer minimal diffs.
- Do not edit files outside the explicitly mentioned scope.
- Do not refactor unless explicitly asked.
- If the requested change appears to require edits in more than 3 files, stop and explain why first.
- Reuse existing patterns; do not introduce new abstractions for small fixes.
- Run only the narrowest relevant tests first.
